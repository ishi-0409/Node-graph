# Node Network Graph Tool

テキストをノードとして整理し、その関係性をネットワーク図として可視化・保存する個人用Webツールです。
Gemini API を使用して、蓄積されたノード間の「隠れた関係性」を自動で推測・提案する機能を搭載しています。

---

## 1. ディレクトリ構成

- `backend/`: FastAPI (Python) バックエンド
  - `main.py`: APIサーバーの本体（Supabase, Gemini APIとの連携を含む）
  - `schema.sql`: Supabase データベースに流し込む SQL スクリプト
  - `.env.example`: 環境変数のテンプレート
- `frontend/`: React + TypeScript + React Flow フロントエンド
  - UI は Vanilla CSS を使用し、モダンなダークテーマ & ガラスモフィズムで構築されています。

---

## 2. セットアップ手順

### ステップ 1: データベース (Supabase) の準備
1. [Supabase](https://supabase.com/) で新規プロジェクトを作成します。
2. Supabase のダッシュボードから **SQL Editor** を開きます。
3. `backend/schema.sql` の内容をコピー＆ペーストし、実行します。
   - これにより、`folders`, `nodes`, `edges` テーブルが作成され、認証なしでAPI通信ができるように RLS が無効化されます。

### ステップ 2: バックエンド (FastAPI) の設定と起動
1. `backend/` ディレクトリ配下に `.env` ファイルを作成します（`.env.example` をコピー）。
2. `.env` 内に以下の値を設定します：
   - `SUPABASE_URL`: Supabase プロジェクトの URL
   - `SUPABASE_KEY`: Supabase プロジェクトの API Key (`anon` キーなど)
   - `GEMINI_API_KEY`: Google AI Studio から取得した Gemini API キー
3. バックエンドの仮想環境をアクティベートし、FastAPI サーバーを起動します。

**Windows (PowerShell):**
```powershell
# 依存関係のインストール (すでに実施済み)
# サーバーの起動
.\backend\venv\Scripts\uvicorn backend.main:app --reload --port 8000
```

### ステップ 3: フロントエンド (React) の起動
1. `frontend/` ディレクトリに移動します。
2. 開発サーバーを起動します。

```powershell
cd frontend
npm run dev
```
起動後、ブラウザで `http://localhost:5173` にアクセスします。

## 3. 主な機能と使い方

1. **フォルダツリー (サイドバー)**
   - フォルダを追加・削除したり、多階層で構築できます。それぞれのフォルダが独立したネットワーク図（キャンバス）を保持します。
2. **ノードの追加と操作**
   - キャンバス上の何もない場所を **ダブルクリック** すると、新規ノードの入力フォームが開きます。
   - 作成したノードはドラッグして自由に配置できます（位置は自動保存されます）。
   - 各ノードをホバーすると、編集や削除のボタンが表示されます。
3. **手動での関係性接続 (エッジ)**
   - ノードの右側にある丸いハンドルから、別のノードの左側のハンドルへドラッグすることで、関係性の線を繋ぐことができます。
   - 接続時に、その関係性を表すラベル（例: 「類似」「対立」など）を任意で入力できます。
   - 繋いだ線をクリックすると、関係性の削除やラベルの編集が可能です。
4. **AIによる関係性の示唆**
   - 右上の「関係性を探す (AI)」ボタンを押すと、フォルダ内のまだ繋がっていないノード群を Gemini がスキャンします。
   - 意味的・文脈的に関連性がありそうなノードのペアと、その理由（ラベル）をカードで提案します。
   - 「承認」を押すと自動でネットワーク図にエッジが追加され、「却下」するとその提案を破棄します。
