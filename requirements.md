# 要件定義

## サービス概要

テキストをノードとして登録し、関係性をネットワーク図で可視化・保存する個人用Webツール。
ノードの選択（何を具体として取り出すか）は人間が行う。関係性はユーザーが手動で繋ぐほか、ノードが蓄積されたときにAIが示唆する。エッジに意味を押し付けず、ユーザーが見たものをそのまま書ける設計とする。

---

## 機能要件

### Must

**フォルダ管理**
- フォルダの作成・名前変更・削除
- フォルダは何階層でも入れ子にできる
- フォルダ自体がネットワーク（ノード・エッジ）を持つ
- サイドバーでフォルダツリーを表示

**ノード**
- テキスト（引用・場面など）を入力して保存
- ノードの編集・削除

**エッジ（関係性）**
- 手動でノード同士を繋ぐ
- ラベルは自由記述（何が似て見えたかをそのまま書く）
- ラベルなしも可（繋がりだけを示す）
- 「関係性を探す」ボタンでGeminiが並置による示唆を提示
- 提示された候補をユーザーが承認/却下してエッジにする

**グラフ表示**
- ネットワーク図として表示
- ノードをドラッグで移動できる

### Nice-to-have

- ノードを別のフォルダにコピーして横断ネットワークを作れる

## 技術スタック・デプロイ構成

| 項目 | サービス | コスト |
|------|---------|--------|
| Frontend | React + React Flow | 無料 |
| Hosting（Frontend） | Firebase Hosting（CDN込み） | 無料 |
| Backend | FastAPI（Python） | 無料 |
| Hosting（Backend） | Google Cloud Run | 無料 |
| DB | Supabase（PostgreSQL、PgBouncer経由） | 無料 |
| AI | Gemini API（無料枠） | 無料 |

## データ構造

```
フォルダ {
  id
  名前
  親フォルダID（nullならルート）
}

ノード {
  id
  フォルダID
  テキスト
  表示位置X
  表示位置Y
}

エッジ {
  id
  フォルダID
  接続元ノードID
  接続先ノードID
  ラベル（任意）
}
```

---

## 使用イメージ

```
本A/                  → 本全体を俯瞰するまとめネットワーク
├── 第1章/            → 第1章の詳細ネットワーク
├── 第2章/            → 第2章の詳細ネットワーク
└── 第3章/            → 第3章の詳細ネットワーク
本B/
└── ...
横断/                 → 複数の本をまたいだノードをコピーして作るネットワーク（Nice-to-have）
```
